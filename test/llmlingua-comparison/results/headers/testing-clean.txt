## Backend Integration Tests

### Setup
Backend integration tests require a running local Supabase instance.

**Prerequisites:**

1. Supabase CLI installed
2. Docker running
3. Environment variables configured

**Setup Steps:**

```bash

# 1. Navigate to API directory
cd /Users/ollorin/igaming-platform/apps/api

# 2. Start local Supabase (uses Docker)
supabase start

# 3. Create .env.test file (if not exists)
cat > .env.test <<EOF
SUPABASE_URL=http://127.0.0.1:54321
SUPABASE_ANON_KEY=[from supabase status]
SUPABASE_SERVICE_ROLE_KEY=[from supabase status]
UPSTASH_REDIS_REST_URL=[your Upstash Redis URL]
UPSTASH_REDIS_REST_TOKEN=[your Upstash Redis token]
DENO_ENV=test
EOF

# 4. Get credentials from Supabase status
supabase status
```text

**Environment Variables:**

- `SUPABASE_URL`: Local Supabase API URL (default: `http://127.0.0.1:54321`)
- `SUPABASE_ANON_KEY`: Public anonymous key for RLS-protected operations
- `SUPABASE_SERVICE_ROLE_KEY`: Admin key for bypassing RLS (use sparingly)
- `UPSTASH_REDIS_REST_URL`: Redis cache URL (required for session tests)
- `UPSTASH_REDIS_REST_TOKEN`: Redis authenticatio…

…

### What's Local-Only
**E2E Tests (NOT in CI):**

E2E tests are **explicitly excluded** from CI for these reasons:

1. **Infrastructure Requirements**: Requires full local Supabase + Redis + frontend servers
2. **Execution Time**: Playwright tests are slow (2-5 minutes per suite)
3. **Flakiness**: Browser automation can be unstable in CI environments
4. **Cost**: GitHub Actions minutes are limited on free tier

## Test Maintenance

### When to Update Tests
1. **API Changes**: Update integration tests when endpoints change
2. **UI Changes**: Update E2E tests when page structure changes
3. **New Features**: Add test coverage for new functionality
4. **Bug Fixes**: Add regression tests to prevent recurrence

### Test Coverage Goals
**Current Coverage:**

- Backend Integration: **100%** endpoint coverage
- E2E Tests: **Core workflows** covered (auth, games CRUD)

**Future Enhancements:**

- Frontend unit tests (component testing with React Testing Library)
- API contract testing (OpenAPI schema validation)
- Performance testing (load testing with k6 or Artillery)
- Visual regression testing (Percy or Chromatic)

### Test Performance
**Backend Tests:**

- Average runtime: ~2 minutes (all suites)
- Parallelization: Not yet implemented (future optimization)

**E2E Tests:**

- Player web: ~1 minute (5 tests)
- Operator web: ~3 minutes (40 tests)
- Headed mode: Add 30% overhead
